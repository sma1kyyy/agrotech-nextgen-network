# техническая документация проекта агротех

## 1. цель и границы проекта
этот стенд показывает как объединить sdn и sd wan для кейса агротех
внутри цод используется sdn под onos
между филиалом и цод используется защищенный wireguard туннель
для сотрудников используется tailscale как удобный вариант удаленного доступа

## 2. роли в команде
team lead отвечает за архитектуру и защиту
network designer отвечает за ip план, onos, ovs и qos
simulation engineer отвечает за docker стенд и сценарии отказа
documentation and budget specialist отвечает за документацию и бюджет

## 3. архитектура и плоскости
control plane
onos управляет ovs1 и ovs2 по openflow 1.3

data plane
трафик приложений идет через ovs1 ovs2 в сети datacenter
трафик филиала идет через frr branch и wireguard

management plane
prometheus и grafana собирают метрики и показывают состояние

## 5. выбранные технологии и обоснование
onos для sdn в цод
ovs для программной коммутации и очередей qos
frr для маршрутизации и failover 4g и спутник
wireguard для безопасного туннеля филиала
tailscale для быстрого доступа сотрудников через nat
prometheus grafana для наблюдаемости
containernet для симуляции сценариев

## 6. ключевые сценарии демонстрации
сценарий 1 автоматическое переключение канала филиала
шаг 1 поднять стенд docker compose up -d
шаг 2 с frr branch проверить два маршрута до 10.0.1.0/24
шаг 3 выключить основной 4g next hop
шаг 4 убедиться что остался маршрут через спутник

сценарий 2 приоритизация трафика
шаг 1 запустить cctv поток и iot поток
шаг 2 проверить очереди ovs и счетчики flow
шаг 3 убедиться что dscp 48 и dscp 34 попадают в приоритетные очереди

## 7. команды проверки
проверка сервисов
docker compose ps

проверка openflow
docker exec -it ovs1 ovs-vsctl show
docker exec -it ovs2 ovs-vsctl show

проверка маршрутов в frr
docker exec -it frr-branch vtysh -c "show ip route"

проверка туннеля
docker exec -it wireguard wg show
docker exec -it frr-branch ping -c 3 10.13.13.1

# сеть для АгроТех

**Комплексное SDN/SD-WAN решение для агропромышленной компании на базе контейнеров Docker**

---

## ТЗ:

Компания "АгроТех" имеет:

1. Центральный офис и ЦОД с высокопроизводительными серверами
2. Удалённый филиал на ферме с IoT-датчиками и камерами видеонаблюдения, который использует два канала связи: 4G и спутниковый
3. Пять удалённых сотрудников, которым нужен доступ к приложениям в ЦОД

Задачи, которые нужно решить:

- обеспечить безопасный доступ к приложениям в ЦОД для филиала и удалённых сотрудников
- приоритезировать трафик видеоконференций и критических IoT-данных с помощью QoS
- обеспечить отказоустойчивость каналов связи филиала, чтобы при падении 4G автоматически включался спутниковый канал

---

## обзор
Сеть разделена на две части: внутри ЦОД используется SDN под управлением контроллера ONOS, который управляет коммутаторами OVS через протокол OpenFlow 1.3, это позволяет централизованно настраивать политики QoS и маршрутизацию трафика. Для подключения удалённого филиала выбран WireGuard, потому что он работает на уровне ядра Linux и даёт минимальные задержки, что важно для потоков с камер видеонаблюдения. Для пяти удалённых сотрудников выбран Tailscale, потому что он автоматически пробивает NAT и не требует ручной настройки каждого клиента. Отказоустойчивость на ферме реализована через FRRouting: демон отслеживает доступность основного 4G-канала и при его падении меняет метрику маршрута, автоматически переключая трафик на спутник. Мониторинг всей инфраструктуры ведётся через связку Prometheus и Grafana, запущенных в Docker. Вся симуляция топологии строится в Containernet, который позволяет запускать Mininet внутри Docker без виртуальных машин.

## архитектура

```
                         ЦОД (Data Center)
          +------------------------------------------------+
          |                                                |
          |   [ONOS Controller]  <-- Control Plane         |
          |         |                                      |
          |    OpenFlow 1.3                                |
          |         |                                      |
          |   [OVS Switch 1] --- [OVS Switch 2]            |
          |    Data Plane           Data Plane             |
          |         |                                      |
          |   [Серверы приложений]  [Prometheus+Grafana]   |
          +--------------------+---------------------------+
                               |
                    WireGuard Tunnel (UDP 51820)
                               |
          +--------------------+---------------------------+
          |        Удалённый филиал (ферма)                |
          |                                                |
          |   [FRR Router]                                 |
          |    /          \                                |
          | [4G]      [Спутник]  <-- резервный канал       |
          |    \          /                                |
          |   [OVS Switch]                                 |
          |       |            |                           |
          |  [IoT-датчики] [CCTV-камеры]                  |
          +------------------------------------------------+

          Удалённые сотрудники (Tailscale mesh)

          [Ноутбук 1] \
          [Ноутбук 2]  \
          [Ноутбук 3]  --  Tailscale Overlay  -->  ЦОД
          [Ноутбук 4]  /
          [Ноутбук 5] /
```

## стек

| Инструмент           | Роль                               | Почему выбран                                                                    |
| -------------------- | ---------------------------------- | -------------------------------------------------------------------------------- |
| ONOS                 | SDN-контроллер в ЦОД               | поддерживает OpenFlow 1.3, есть готовый Docker-образ                             |
| Open vSwitch (OVS)   | программный коммутатор Data Plane  | управляется ONOS через OpenFlow, поддерживает DSCP и очереди QoS                 |
| WireGuard            | SD-WAN туннель для филиала         | работает в ядре Linux, минимальные задержки, подходит для видеопотоков           |
| Tailscale            | SD-WAN оверлей для сотрудников     | автоматический NAT traversal, не нужна ручная настройка на каждом устройстве     |
| FRRouting (FRR)      | маршрутизация и failover в филиале | поддерживает отслеживание BFD/статических маршрутов для автопереключения каналов |
| Containernet         | симуляция сетевой топологии        | запускает Mininet внутри Docker, не нужны виртуальные машины                     |
| Prometheus + Grafana | мониторинг и метрики               | стандартная связка, оба инструмента запускаются в Docker                         |
| Docker Compose       | оркестрация всех сервисов          | единая точка запуска всего стенда без VM                                         |

## быстрый старт

```bash
#1 клонировать репозиторий
git clone https://github.com/agrotech-sdn/network-project.git
cd network-project

#2 загрузить ядерный модуль OVS (требуется на хост-машине)
sudo modprobe openvswitch

#3 запустить весь стенд
sudo docker compose up -d

#4 проверить, что все сервисы запустились
docker compose ps

#5 открыть веб-интерфейс ONOS
# http://localhost:8181/onos/ui  (логин: onos, пароль: rocks)
# Grafana: http://localhost:3000  (логин: admin, пароль: admin)
```
